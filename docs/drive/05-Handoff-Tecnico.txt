1. STACK
========

Frontend:
- Next.js 14 (App Router)
- TypeScript strict
- Tailwind CSS 3 (con tema custom en tailwind.config.ts)
- Recharts (gráficos)
- Lucide React (iconos)
- Sonner (toasts)
- Zustand (estado global con persist)

Backend:
- Supabase (managed Postgres + Auth + Storage + Realtime)
- @supabase/ssr para SSR de Next.js
- @supabase/supabase-js v2.101+

Deploy:
- Vercel auto-deploy desde rama main
- Variables de entorno en Vercel Project Settings

3. PATRONES CLAVE
=================

A. SERVER COMPONENTS para fetch inicial:
   src/app/(dashboard)/cobros/page.tsx hace fetch con:
     const supabase = await createClient();
     const { data: cobros } = await supabase.from('cobros').select(...);
   Después pasa data al Client Component como prop.

B. CLIENT COMPONENTS para interactividad:
   - 'use client' al inicio
   - useAuthStore() para perfil
   - createClient() de lib/supabase/client para mutaciones
   - services.ts centraliza CRUD

C. SYNC LIBRO DIARIO:
   - Cada cobro/venta/factura/instalación pagada → entrada en libro_diario
   - origen_id + origen_tipo permiten trazar
   - Borrar entrada → revierte source O elimina source (depende del tipo)

D. ACTIVITY LOG:
   - logActivity() en services.ts
   - Llamado en cada CRUD automáticamente
   - Página /historial filtra por usuario/entidad/acción

E. RLS (Row Level Security):
   - Habilitado en TODAS las tablas
   - Política "Gestionar X" típica para todos los autenticados
   - libro_diario tiene política especial "ver_libro_diario" para SELECT

F. DESIGN SYSTEM:
   - Variables de Tailwind en tailwind.config.ts
   - Clases reutilizables en globals.css:
     · .card, .stat-card, .kpi-card
     · .btn-primary, .btn-secondary, .btn-danger
     · .badge, .badge-success, .badge-warning, etc.
     · .modal-overlay, .modal-content (bottom-sheet en móvil)
     · .list-card (para vista mobile de listas)

5. AUTH FLOW
============

1. Usuario va a /login
2. Email + password → Supabase Auth
3. Si OK: redirect a /dashboard
4. Middleware valida sesión en cada request
5. AuthProvider carga el profile (con rol y equipo)
6. Sidebar filtra menú según rol


6. ROLES
========

admin            → todo
financiero       → cobros, facturas, ventas, libro diario, finanzas, reportes, gastos
administrativo   → inventario, conciliacion, revendedores, equipo
soporte          → clientes, instalaciones, tareas


7. DEPLOY & CI/CD
=================

Deploy automático:
- Push a rama main → Vercel detecta cambio
- Build de Next.js + lint
- Deploy en ~2 minutos

Variables de entorno (Vercel Project Settings):
- NEXT_PUBLIC_SUPABASE_URL
- NEXT_PUBLIC_SUPABASE_ANON_KEY (puede ser legacy o publishable)

Migraciones SQL:
- Manualmente: pegar en Supabase SQL Editor
- Programáticamente: scripts/run-migration.mjs con SUPABASE_PAT


8. VERSIONADO
=============

Semantic Versioning (MAJOR.MINOR.PATCH):
- PATCH: bug fixes (v1.4.1, v1.4.2)
- MINOR: nuevas features retro-compatibles (v1.5.0)
- MAJOR: breaking changes (v2.0.0)

Cada release:
1. Bump en package.json y src/lib/version.ts
2. Update CHANGELOG.md
3. git tag -a vX.Y.Z -m "..."
4. git push --tags

Scripts npm:
- npm run release:patch
- npm run release:minor
- npm run release:major


9. ENVIRONMENT LOCAL
====================

Setup:
1. git clone https://github.com/dopcoin/ZOE-Net-Gesti-n.git
2. cd ZOE-Net-Gesti-n
3. npm install
4. cp .env.example .env.local  (rellenar con valores de Supabase)
5. npm run dev → localhost:3000

Variables mínimas:
NEXT_PUBLIC_SUPABASE_URL=https://....supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJ...

Crear usuario admin:
1. Supabase Dashboard → Auth → Users → Add User
2. Después en SQL Editor:
   INSERT INTO profiles (id, email, nombre, apellido, rol, equipo, activo)
   VALUES ('UUID-FROM-AUTH', 'email@example.com', 'Nombre', 'Apellido',
           'admin', 'administrativo', true);


10. PROBLEMAS COMUNES Y SOLUCIONES
==================================

ERROR: "fecha_retiro column not found"
SOLUCIÓN: Correr supabase/migrations/EJECUTAR_AHORA.sql

ERROR: Cliente no aparece en /cobros
CAUSA: estado != 'activo'/'becado'/'nuevo'
SOLUCIÓN: Editar cliente, cambiar estado

ERROR: Stock negativo después de venta
CAUSA: Trigger de descuento de stock no se ejecutó
SOLUCIÓN: Verificar manualmente y editar producto

ERROR: WhatsApp no abre
CAUSA: Cliente no tiene teléfono registrado
SOLUCIÓN: Agregar teléfono en su ficha

ERROR: 401 Unauthorized en cualquier query
CAUSA: ANON key inválida o expirada
SOLUCIÓN: Rotar key en Supabase y actualizar Vercel
